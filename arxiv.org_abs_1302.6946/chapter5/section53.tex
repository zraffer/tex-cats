\subsection{\caseENGRUS{Monads formalize context}{ / }{ĞœĞ¾Ğ½Ğ°Ğ´Ñ‹ Ñ„Ğ¾Ñ€Ğ¼Ğ°Ğ»Ğ¸Ğ·ÑƒÑÑ‚ ĞºĞ¾Ğ½Ñ‚ĞµĞºÑÑ‚}}

\begin{blockENG}
Monads can formalize assumptions about the way one will do business throughout a domain. For example, suppose that we want to consider functions that do not have to return a value for all inputs.\index{monad!formalizing context}\index{context} Such {\em partial functions}\index{partial functions} can be composed. Indeed, given a partial function $f\colon A{â†’} B$ and a partial function $g\colon B{â†’} C,$ one gets a partial function $g\circ f\colon A{â†’} C$ in an obvious way.
\end{blockENG}

\begin{blockRUS}
\end{blockRUS}

\begin{blockENG}
Here we are drawing arrows as though we are talking about functions, but there is an implicit context in which we are actually talking about partial functions. Monads allow us to write things in the â€œfunctionalâ€ way while holding the underlying context. What makes them useful is that the notion of {\em context} we are using here is made formal.
\end{blockENG}

\begin{blockRUS}
\end{blockRUS}

\begin{exampleENG}[Partial functions]\label{ex:partial function monad}\index{a monad!partial functions}\index{partial function}\index{a monad!maybe}
Partial functions can be modeled by ordinary functions, if we add a special â€œno answerâ€ element to the codomain. That is, the set of partial functions $A{â†’} B$ is in one-to-one correspondence with the set of ordinary functions $A{â†’} B\sqcup{\{â˜º\}}.$ For example, suppose we want to model the partial function $f(x){\coloneqq}\frac{1}{x^2-1}\colon{â„}{â†’}{â„}$ in this way, we would use the function 
$$f(x){\coloneqq}\begin{cases}
\frac{1}{x^2-1}&\tn{if } x\neq -1 \tn{ and } x\neq 1,\\
{â˜º}&\tn{if } x=-1,\\
{â˜º}&\tn{if } x= 1.
\end{cases}
$$
An ordinary function $f\colon A{â†’} B$ can be considered a partial function because we can compose with the inclusion 
\begin{align}\label{dia:first eta}
B{â†’} B\sqcup{\{â˜º\}}
\end{align}

But how do we compose two partial functions written in this way? Suppose $f\colon A{â†’} B\sqcup{\{â˜º\}}$ and $g\colon B{â†’} C\sqcup{\{â˜º\}}$ are functions. First form a new function 
\begin{align}\label{dia:first monad}
g'{\coloneqq}g\sqcup{\{â˜º\}}\colon B\sqcup{\{â˜º\}}{â†’} C\sqcup{\{â˜º\}}\sqcup{\{â˜º\}}
\end{align}
then compose to get $(g'\circ f)\colon A{â†’} C\sqcup{\{â˜º\}}\sqcup{\{â˜º\}},$ and finally send both ${â˜º}$'s to the same element by composing with 
\begin{align}\label{dia:first mu}
C\sqcup{\{â˜º\}}\sqcup{\{â˜º\}}{â†’} C\sqcup{\{â˜º\}}.
\end{align}

What does this mean? Every element $a\in A$ is sent by $f$ to either an element $b\in B$ or â€œno answerâ€. If it has an answer $f(a)\in B,$ this is either sent by $g$ to an element $g(f(a))\in C$ or to â€œno answerâ€. We get a partial function $A{â†’} C$ by sending $a$ to $g(f(a))$ if possible or to â€œno answerâ€ if it gets stopped along the way.

This monad is sometimes called the {\em maybe monad} in computer science, because a partial function $f\colon A{â†’} B$ takes every element of $A$ and either outputs just an element of $B$ or outputs nothing; more succinctly, it outputs a â€œmaybe $B$â€.
\end{exampleENG}

\begin{definitionENG}[Monad]\label{def:monad}\index{monad}\index{monad!on $\Set$}
A {\em monad on $\Set$} is defined as follows: One announces some constituents (A. functor, B. unit map, C. multiplication map) and asserts that they conform to some laws (1. unit laws, 2. associativity law). Specifically, one announces
\begin{enumerate}[A.]
\item a functor $T\colon\Set{â†’}\Set,$
\item a natural transformation $\eta\colon\id_{\Set}{â†’} T,$ and 
\item a natural transformation $\mu\colon T\circ T{â†’} T$
\end{enumerate}
We sometimes refer to the functor $T$ as though it were the whole monad; we call $\eta$ the {\em unit map} and we call $\mu$ the {\em multiplication map}. One asserts that the following laws hold:
\begin{enumerate}[1.]
\item The following diagrams of functors $\Set{â†’}\Set$ commute:
\begin{align*}
\xymatrix@=30pt{T\circ\id_\Set\ar[r]^-{\id_T\diamond\eta}\ar[dr]_{=}&T\circ T\ar[d]^\mu\\&T}\hspace{1in}
\xymatrix@=30pt{\id_\Set\circ T\ar[r]^-{\eta\diamond\id_T}\ar[dr]_=&T\circ T\ar[d]^\mu\\&T}
\end{align*}
\item The following diagram of functors $\Set{â†’}\Set$ commutes:
\begin{align*}
\xymatrix@=30pt{T\circ T\circ T\ar[r]^-{\mu\diamond\id_T}\ar[d]_-{\id_T\diamond\mu}&T\circ T\ar[d]^\mu\\T\circ T\ar[r]_\mu&T}\end{align*}
\end{enumerate}
\end{definitionENG}

\begin{exampleENG}[List monad]\label{ex:monad}\index{a monad!List}
We now go through Definition~\ref{def:monad} using what is called the $\List$ monad.\index{list} The first step is to give a functor $\List\colon\Set{â†’}\Set,$ which we did in Example~\ref{ex:free monoid}. As a monad, the functor $\List$ comes with two natural transformations, a unit map $\eta$ and a multiplication map $\mu.$ Given a set $X,$ the unit map $\eta_X\colon X{â†’}\List(X)$ returns singleton lists as follows
$$\xymatrix@=.5pt{

\ar@{..}[rrr]+<.3in,0pt>&&&\\\\\\

&q\ar@{|->}[rr]&&[q]\\

$$\xymatrix@=.5pt{

\ar@{..}[rrr]+<.6in,0pt>&&&\\\\\\

$$\xymatrix@=30pt{

\vspace{.1in}
$$\xymatrix@=30pt{
\Big[\big[[p, q], [r]\big], \big[[], [r, q, q]\big]\Big]\ar@{|->}[r]^-{\mu\circ\id_{\List}}\ar@{|->}[d]_{\id_{\List}\circ\mu}&\big[[p, q], [r], [], [r, q, q]\big]\ar@{|->}[d]^\mu\\\big[[p, q, r], [r, q, q]\big]\ar@{|->}[r]_\mu&[p, q, r, r, q,q]

\end{blockENG}

\begin{definitionENG}\label{def:kleisli}\index{category!Kleisli}\index{Kleisli category}\index{monad!Kleisli category of}
Let ${â†’}p=(T,\eta,\mu)$ be a monad on $\Set.$ Form a new category, called the {\em Kleisli category for ${â†’}p$}, denoted $\Kls({â†’}p),$ with sets as objects, $\Ob(\Kls({â†’}p)){\coloneqq}\Ob(\Set),$ and with $$\Hom_{\Kls({â†’}p)}(X,Y){\coloneqq}\Hom_\Set(X,T(Y))$$ for sets $X,Y.$ The identity morphism $\id_X\colon X{â†’} X$ in $\Kls({â†’}p)$ is given by $\eta\colon X{â†’} T(X)$ in $\Set.$ The composition of morphisms $f\colon X{â†’} Y$ and $g\colon Y{â†’} Z$ in $\Kls({â†’}p)$ is given as follows. Writing them as functions, we have $f\colon X{â†’} T(Y)$ and $g\colon Y{â†’} T(Z).$ The first step is to apply the functor $T$ to $g,$ giving $T(g)\colon T(Y){â†’} T(T(Z)).$ Then compose with $f$ to get $T(g)\circ f\colon X{â†’} T(T(Z)).$ Finally, compose with $\mu_Z\colon T(T(Z)){â†’} T(Z)$ to get the required function $X{â†’} T(Z).$ The associativity of this composition formula follows from the associativity law for monads.
\end{definitionENG}

\begin{definitionRUS}\label{def:kleisli}\index{category!Kleisli}\index{Kleisli category}\index{monad!Kleisli category of}
\end{definitionRUS}

\begin{exampleENG}
Recall the monad ${â†’}p$ for partial functions, $T(X)=X\sqcup{\{â˜º\}},$ from Example~\ref{ex:partial functions as monad}. The Kleisli category $\Kls({â†’}p)$ has sets as objects, but a morphism $f\colon X{â†’} Y$ means a function $X{â†’} Y\sqcup{\{â˜º\}},$ i.e a partial function. Given another morphism $g\colon Y{â†’} Z,$ the composition formula in $\Kls({â†’}p)$ ensures that $g\circ f\colon X{â†’} Z$ has the appropriate behavior.

Note how this monad allows us to make explicit our assumption that all functions are partial, and then hide it away from our notation.

\begin{remarkENG}\label{rem:ordinary are kleisli}
For any monad ${â†’}p=(T,\eta,\mu)$ on $\Set,$ there is a functor $i\colon \Set{â†’}\Kls({â†’}p)$ given as follows. On objects we have $\Ob(\Kls({â†’}p))=\Ob(\Set),$ so take $i=\id_{\Ob(\Set)}.$ Given a morphism $f\colon X{â†’} Y$ in $\Set,$ we need a morphism $i(f)\colon X{â†’} Y$ in $\Kls({â†’}p),$ i.e. a function $i(f)\colon X{â†’} T(Y).$ We assign $i(f)$ to be the composite $X\To{f}Y\To{\eta}T(Y).$ The functoriality of this mapping follows from the unit law for monads.

The point is that any ordinary function (morphism in $\Set$) has an interpretation as a morphism in the Kleisli category of any monad. More categorically, there is a functor $\Set{â†’}\Kls({â†’}p).$
\end{remarkENG}

\begin{exampleENG}\label{ex:preorder monad}
Let $A$ be any preorder. We speak of $A$ throughout this example as though it was the linear order given by time because this is a nice case, however the mathematics works for any $A\in\Ob(\PrO).$ 

There is a monad ${â†’}p=(T,\eta,\mu)$ that captures the idea that a function $f\colon X{â†’} Y$ occurs in the context of time in the following sense: The output of $f$ is determined not only by the element $x\in X$ on which it is applied but also by the time at which it was applied to $x$; and the output of $f$ occurs at another time, which is not before the time of input.

The functor part of the monad is given on $X\in\Ob(\Set)$ by
$$T(X)=\{p\colon A{â†’} A\times X{\;|\;}\tn{ if } p(a)=(a',x)\tn{ then } a'\geq a\}.$$

If for every $a\in A$ you have a later element $a'\geq a$ and a function $p\colon A{â†’} A\times X$ that takes elements of $A$ to later elements of $A$ and values of $X,$ then $p(a')$ is a still later element of $A$ and a value of $X,$ as desired.

Morphisms in the Kleisli category $\Kls({â†’}p)$ can be curried to be functions $f\colon A\times X{â†’} A\times Y$ such that if $f(a,x)=(a',y)$ then $a'\geq a.$
